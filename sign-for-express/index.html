Introduction

This Adobe Express add-on provides a simple integration with Adobe Acrobat Sign, to simplify the sending of an Express asset for e-signature. It makes use of the following capabilities:

  • Spectrum Web Components for the User Interface
  • OAuth 2.0 support provided by Adobe Express
  • PDF Rendition support provided by Adobe Express

The add-on assumes that the user has pre-existing credentials to access Adobe Acrobat Sign and are permitted to create a new agreement.

Usage

Please note that, as the Partner API Application used within Adobe Acrobat Sign is not currently certified, you will need to enable the following entry within the Security Settings of your account for the add-on to function correctly.

  • Allow Uncertified partner applications to access data from this account

Once you have access to the add-on it appears as:

The add-on provides a step-by-step flow to allow the user to:

  • Authorize access to their Acrobat Sign account, by clicking on the Authorize button to launch the OAuth 2.0 flow (check your browser, to ensure that pop-up windows are permitted)

Once credentials have been successfully entered, the user will be prompted to Allow Access

Following consent, the user can choose to render either the Current Page or Whole Document to PDF which, when completed, displays a button that allows the user to Launch Acrobat Sign.

When the user clicks on the button, it will:

  • Upload the PDF rendition to the Sign platform
  • Create a draft agreement using the uploaded document
  • Open the draft agreement in a separate browser tab/window within which the user can review, adjust, and send the agreement as appropriate
